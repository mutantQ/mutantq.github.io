# 조절 가능한 하이퍼파라미터들

# $\epsilon$

### 이름

엡실론 (그리스 알파벳)

### 범위

$0\le\epsilon\le1$, $\epsilon$은 실수

### 값의 의미

- $\epsilon=0$: 완전한 균등 랜덤 행동
- $\epsilon=1$: 완전한 최적 행동
(단, 여기서 “최적 행동”은 에이전트가 생각하는 최적 행동을 뜻함)

### 통상적인 값

- $\epsilon=0.9$, $\epsilon=0.99$

### 예시

- $\epsilon=0.1$이고 $\{위,아래,오른쪽,왼쪽\}$ 중 최적 행동이 $위$인 경우.

# $\gamma$

### 이름

감마 (그리스 알파벳)

### 범위

$0\le \gamma \le 1$, $\gamma$는 실수

### 값의 의미

- $\gamma=0$: 완전히 단기적인 최적 행동 전략 탐색
- $\gamma=1$: 완전히 장기적인 최적 행동 전략 탐색
(단기적 보상과 장기적 보상의 가중치가 동일함)

### 통상적인 값

- $\gamma=0.9$, $\gamma=0.99$

### 예시

- $n=\infty$일 때. $n$에 대한 설명은 추후에 계속.
    
    $(누적\;보상)=(지금\;보상)+\gamma(다음\;보상)+\gamma^2(다음다음\;보상)+\cdots$
    

## $n$

### 이름

엔 (영문 알파벳)

### 범위

$n$은 자연수 $(n=1,2,3\cdots)$

### 값의 의미

- $n=1$: 지금 보상 이후의 누적 보상은 예측치으로 대체
- $n=k$: 처음 $k$개 보상에 대한 누적 보상을 제외한 남은 보상은 예측치로 대체
- $n=\infty$: 예측치를 사용하지 않고 전부 실제 보상을 사용

### 통상적인 값

- $1 \le n \le 10.$ 보통의 경우 $10$이 성능 측면에서 최적임.
- 연산량을 줄이기 위해 $n$을 작은 값으로 설정하기도 함.

### 예시

- $n=3$인 경우
    
    $(누적\;보상)=(보상)_0+\gamma(보상)_1+\gamma^2(보상)_2+(예측치)$
